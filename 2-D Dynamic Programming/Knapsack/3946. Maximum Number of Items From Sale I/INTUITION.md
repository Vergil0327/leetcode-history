正確的解題直覺：把問題拆解為「買第一件」與「自由加購」

為了解決買第一件和買多件收益不同的問題，我們可以把決策分成兩個階段：

1. 階段一：決定你要「啟動」哪些商品的贈送功能（這是一個 0/1 背包問題）。
   - 我們選擇一個商品集合 $S$，並且對 $S$ 中的每種商品都剛好只買 1 件。
   - 此時，我們付出的總成本是：$\sum_{i \in S} \text{price}_i$。
   - 我們能獲得的總件數是：$|S|$（買的） + 所有由 $S$ 延伸出去的免費贈品總數。
2. 階段二：利用剩餘的預算，進行完全背包的「自由加購」。
   - 一旦我們決定了要買哪幾種商品的第一件，剩下的預算（$\text{budget} - \text{cost}$）我們可以用來無限量加購任何商品。
   - 因為這時候贈送功能都已經定案了，加購任何商品 i 的收益純粹就是固定不變的 +1。

因為這題的預算非常小（$\text{budget} \le 1500$），這暗示了我們可以預先算好完全背包的超值加購表，然後再用動態規劃（DP）去窮舉那些「買第一件」的組合！

正確的 3 步驟演算法

步驟 1：預處理完全背包加購表我們先不管任何贈送規則。如果我們有預算 $b$，純粹拿去買商品，最多能買幾件？（每買一件收益就是 1）。我們可以用完全背包求出一個一維陣列 pure_knap[b]，代表用預算 $b$ 自由加購，最多能得到幾件商品。

步驟 2：計算每種商品的「第一件真正總收益」如果我們打算購買商品 i 的第一件，那我們必須付出 price_i 的代價。此時，除了得到商品 i 本身之外，還會獲得所有滿足 factor_j % factor_i == 0 (j != i) 的免費商品 j。因此，商品 i 作為「第一件」時的真正總件數收益為：gain[i] = 1 + (有多少個商品 j 滿足 factor_j 是 factor_i 的倍數且 j != i)

步驟 3：0/1 背包結合自由加購我們對這 $n$ 個商品的 gain[i] 與 price_i 跑一次 0/1 背包。定義 dp[b]：代表我們花費預算 $b$，挑選某些商品並「各買 1 件」時，能獲得的最大總件數（含贈品）。最後，我們窮舉花在第一階段（買第一件）的預算 $b$：總體最大收益 = max( dp[b] + pure_knap[budget - b] )
